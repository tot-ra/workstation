# Управление задачами

## Создание задачи
При работе над задачей создай `{TASK-ID}.md`:
- Task ID: из jira/git branch, или придумай сам
- Путь: `~/git/mind/agent/tasks/`

## Структура файла задачи
```markdown
## Why
Описание проблемы

## How
Внесённые изменения

## TODO
- [ ] Подзадача 1
- [ ] Подзадача 2
```

## Процесс работы
1. Приоритизируй задачи, начинай с высокоприоритетных
2. Отмечай `[x]` выполненное
3. Обновляй файл перед тестами или завершением
4. `git commit` когда готово

## Команды требующие sudo
1. Создай скрипт с командами
2. Попроси пользователя запустить
3. Читай логи из файла
4. Пример: `make build &> ../tmp.log`
