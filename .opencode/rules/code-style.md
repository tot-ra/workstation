# Стиль кода

## Общие принципы
- Пиши чистый, лаконичный код
- Проверяй похожий код рядом для консистентности
- Фокусируйся на текущей задаче, не рефактори несвязанный код

## Комментарии
- НЕ ПИШИ КОММЕНТАРИИ в коде
- Вместо `// TODO ...` добавляй в TODO.md

## Тесты
- Разбивай на под-тесты, избегай больших тестов
- Не используй параметрические тесты - отдельные под-тесты лучше
- Используй комментарии-секции:
```
// ARRANGE
// ACT
// ASSERT
```
